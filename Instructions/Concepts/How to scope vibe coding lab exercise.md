# 如何限定 Vibe 编码实验室练习的范围

应将 Vibe 编码实验室练习的范围限定为侧重于特定任务和结果，确保参与者可以在有限的时间范围内有效地学习和应用这些概念。 

## Vibe 编码项目类型

实验室练习的编码项目需要仔细地限定范围（狭窄或特定的范围），以适应 20-30 分钟以上的练习。

1. 从头开始项目（实现基础知识、存根功能、页面导航，但应用数据很少或没有应用数据）

    1. 创建原型电子商务应用

        示例提示：

        - 我需要使用 JavaScript、HTML 和 CSS 创建原型电子商务应用。
        - 原型电子商务应用必须提供动态用户界面，该界面自动缩放以适应在桌面和手机设备上的查看。
        - 电子商务应用应包含以下页面：产品列表、产品详细信息、购物车和结帐。 每个页面应该提供基本的功能和页面之间的前进/后退导航。
        - 使用 10 种水果产品的简单数据集。 包括：产品名称、说明、单价（单位可以是商品数量或盎司、磅等）和代表产品的简单图像。
        - “产品列表”页面应显示包含基本信息（例如产品名称、单价和图像）的产品列表。 “产品列表”页面还应提供选择产品数量的方法，以及向购物车添加所选商品的选项。
        - “产品详细信息”页面应显示有关所选产品的详细信息，包括产品名称、说明、单价和图像。 “产品详细信息”页面还应提供导航回“产品列表”页面的方法。
        - “购物车”页面应显示添加到购物车的产品的列表，包括产品名称、数量和总价。 “购物车”页面还应提供用于更新购物车中产品的数量，并从购物车中删除产品的方法。
        - “结帐”页面应显示购物车中产品的摘要，包括产品名称、数量和总价。 “结帐”页面还应提供确认订单的方法。 
        - 原型电子商务应用应提供页面之间的基本导航。 在“产品列表”、“产品详细信息”、“购物车”和“结帐”页面的左侧创建一个可折叠导航栏。 该导航栏应使用户能够在“产品列表”、“产品详细信息”、“购物车”和“结帐”页面之间导航。
        - 原型电子商务应用应具有基本的样式，使其在视觉上具有吸引力，但它不需要具备完全可响应性或精美的设计。
        - 原型电子商务应用不会包括任何后端功能，例如用户身份验证、付款处理或数据库集成。 它将是一个静态原型，演示电子商务应用的基本概念。

    1. 为 AI 增强型应用创建原型（使用 Python 和 Hugging Face 模型）

1. 向项目添加新功能

    1. 向现有应用添加新功能
        - 例如，向电子商务应用添加搜索功能。
        - 搜索功能应使用户能够按名称或说明搜索产品。
        - 搜索结果应显示与搜索条件匹配的产品列表，包括产品名称、单价和图像。
        - 搜索功能应可从“产品列表”页面访问，并且应提供导航回“产品列表”页面的方法。

    1. 向现有应用添加新页面

        - 例如，向电子商务应用添加“联系我们”页面。
        - “联系我们”页面应包含一个表单，使用户能够提交其姓名、电子邮件地址和电子邮件。
        - “联系我们”页面还应提供导航回“产品列表”页面的方法。

1. 更改工具、框架或技术项目

    1. 更改现有应用的前端框架

        - 例如，将电子商务应用中的前端框架从 JavaScript 更改为 React 或 Angular。
        - 新的前端框架应提供与原始应用相同的功能，包括产品列表、产品详细信息、购物车和结帐。
        - 新的前端框架还应提供动态用户界面，该界面自动缩放以适应在桌面和手机设备上的查看。

    1. 更改现有应用的后端技术

        - 例如，将电子商务应用的后端技术从 Node.js 更改为 Python Flask 或 Django。
        - 新的后端技术应提供与原始应用相同的功能，包括用户身份验证、付款处理和数据库集成。

1. 代码评审和改进项目

    1. 向项目添加日志记录
    1. 向项目添加单元测试

1. 编码语言转换项目

    将现有应用转换为其他语言或框架。

    例如，将 Python 应用转换为 Java 应用或 C# 应用。
